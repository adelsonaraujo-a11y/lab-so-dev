# 1. A apresentação mostrou que "instruções de I/O são privilegiadas". 
#    Com base no que o strace revelou, explique com suas palavras como um programa
#    de usuário consegue fazer I/O sem executar instruções privilegiadas diretamente.

Resposta 1: Ele consegue fazer isso ao chamar funções como o write() que são syscalls.
Essas chamadas mudam o usuario para o modo kernel de forma controlada executando I/O com
segurança, sendo que o strace mostra exatamente esse processo
